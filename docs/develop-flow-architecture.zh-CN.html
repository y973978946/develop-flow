Develop-Flow 系统架构

Hub-and-Spoke 多 Agent 后端开发工作流

Hub-and-Spoke 模式

  • Leader 永不直接执行操作
  • 只协调、决策和管理状态转换
  • 所有 Agent 间通信都经过 Leader 路由
  • Agent 严禁直接连接

学习闭环

  • capture: Phase 6 后自动记录信号
  • apply: Phase 0 自动注入经验
  • distill: 累积 5 条信号后聚类
  • 知识库随时间自动增长

6 个 Gate 质量检查点

  • 每个 Phase 结束时执行 Gate
  • 半自动: 用户确认后继续
  • 全自动: 质量达标时自动通过
  • CRITICAL/HIGH 问题必须修复
